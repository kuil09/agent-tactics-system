# Decision Record: 제품화 실행 소유권을 CTO에 고정

- Date: 2026-04-02
- Owner: CEO
- Related issues: [NIT-66](/NIT/issues/NIT-66)

## Context

[NIT-66](/NIT/issues/NIT-66)은 현재 저장소를 바탕으로 "실험 결과를 실제 제품 구현으로 전환"하라는 요청이다.

현재 `agent-tactics-system` 저장소는 참조 런타임(reference runtime, 운영 원리를 재현하는 기준 구현), 검증 인계, 승인 차단, 복구 계약까지는 갖췄다. 반면 실제 제품화는 외부 제공자 연결, 승인 흐름 연결, 복구 범위 확장, 운영 절차 정리처럼 연속적인 기술 실행과 추가 분해가 필요한 단계다.

이 단계는 CEO가 직접 구현하는 것보다 CTO가 기술 실행 소유권을 일관되게 갖고, 필요 시 Founding Engineer와 PMO로 다시 분해하는 편이 대기와 판단 중복을 줄인다.

## Decision

1. `agent-tactics-system`의 실제 제품화 실행 소유권은 CTO가 가진다.
2. CEO는 범위 잠금, 우선순위, 승인 경계만 정하고 직접 구현 지시는 하지 않는다.
3. CTO는 현재 저장소 기준으로 제품화 실행을 단계적으로 분해하고, 각 단계의 완료 기준과 위험을 이슈 체인(issue chain, 연결된 작업 흐름)으로 남긴다.
4. 제품화 과정에서 권한 경계나 운영 절차를 바꾸면, CTO는 새 이슈와 결정 기록을 남겨 이사회가 추적 가능하게 만든다.

## Why

- 현재 병목은 아이디어 부족이 아니라 실행 소유권의 분산이다.
- 실제 제품화는 코드 수정, 검증, 통합, 운영 연결을 연속해서 다뤄야 하므로 기술 책임자가 한 축으로 잡는 편이 빠르다.
- CEO는 전략과 승인 경계에 집중하고, CTO는 기술 실행과 분해를 맡는 구조가 재작업을 줄인다.

## Consequences

- [NIT-66](/NIT/issues/NIT-66)은 CEO의 위임 결정과 추적 경로 고정이 끝나면 종료할 수 있다.
- CTO는 후속 실행 이슈를 받아 제품화 착수 범위, 첫 단계 산출물, 후속 분해 계획을 댓글로 남겨야 한다.
- 이후 제품화 지연이나 운영 결함은 우선 CTO 소유 이슈에서 추적하고, CEO는 승인 또는 우선순위 조정이 필요한 경우에만 개입한다.
